what is Terraform?
Terraform is an Infrastructure as Code (IaC) tool developed by HashiCorp.

Instead of manually creating resources in AWS Console, Azure Portal, or GCP Console, you write code that defines your infrastructure.

For example, rather than:

Logging into AWS
Clicking EC2
Launch Instance
Selecting AMI
Choosing instance type
Configuring security groups

You can write:

resource "aws_instance" "web" {
  ami           = "ami-xxxxxxxx"
  instance_type = "t2.micro"
}

and run:

terraform apply

Terraform will create the EC2 instance automatically.

Why Terraform is required?

Imagine you're a Linux Administrator managing 50 servers.

Without Terraform

Every time you need infrastructure:

Login to AWS Console
Create VPC
Create Subnets
Create Security Groups
Create EC2
Attach Volumes
Configure Networking

Problems:

❌ Time-consuming
❌ Human errors
❌ Difficult to reproduce
❌ No version control
❌ No automation

Complete Terraform Lifecycle

Write Code
     ↓
terraform init
     ↓
terraform validate
     ↓
terraform plan
     ↓
terraform apply
     ↓
Resources Created
     ↓
Modify Code
     ↓
terraform plan
     ↓
terraform apply
     ↓
Resources Updated
     ↓
terraform destroy
     ↓
Resources Deleted

These are Terraform-generated and project-related files:
File
Purpose
Should You Keep It?
main.tf
Your Terraform configuration file (infrastructure code).
✅ Yes
steps.md
Markdown documentation file. Usually contains project steps, notes, commands, or instructions.
✅ Yes
myplan
Binary execution plan generated by terraform plan -out=myplan. It stores exactly what Terraform will create/change.
⚠️ Temporary
terraform.tfstate
Terraform state file. Tracks all resources Terraform manages and their current IDs, IPs, attributes, etc.
✅ Very Important
terraform.tfstate.backup
Automatic backup of the previous state file. Terraform creates it before modifying state.
